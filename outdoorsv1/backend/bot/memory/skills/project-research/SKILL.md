# Project Research Skill

## Use this skill when
- User asks to "research" or "find everything about" a project/topic
- Need comprehensive information gathering across multiple sources
- Building understanding of an existing project or initiative

## Research methodology

### 1. Multi-source search strategy
**Email sources**: Search across ALL relevant email accounts (not just primary)
- Baret Scholars account for Baret-related projects
- Rice account for academic projects  
- Personal accounts for personal projects

**Cloud storage**: Check shared folders and personal drives
- Look for project folders, shared documents, rulebooks, specs
- Check for "Current Tasks", planning docs, communication logs

**Local filesystem**: Search project directories
- Check `C:\Users\anton\Documents\Projects\` for existing implementations
- Look for source code, documentation, assets

### 2. Information organization
For complex projects, create focused memory files instead of monolithic docs:
- **overview.md**: Context, key people, timeline, business purpose
- **[project]-rules.md**: Game rules, specifications, requirements  
- **tech-stack.md**: Implementation details, architecture, database schema

### 3. Known limitations
**WhatsApp search**: Chrome MCP frequently times out on WhatsApp Web. For WhatsApp research, recommend manual check or ask user to search specific conversations.

### 4. Search patterns
- Use broad terms initially ("Fragile World", project name)
- Then narrow to specific aspects (rules, deadlines, technical specs)
- Cross-reference findings between sources to build complete picture

## Success criteria
- Found information across multiple sources (not just one)
- Organized findings into actionable, referenceable memory
- Identified key people, documents, and next steps
- Clear summary of what was found vs. what requires manual follow-up
